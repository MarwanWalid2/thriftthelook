# ThriftTheLook Winning Video Implementation Plan

> **For agentic workers:** Record only observable product behaviour. Never call synthetic inventory live, never show a secret, and never claim a model/provider ran unless that take proves it.

**Goal:** Produce a public, under-three-minute, audio-led Build Week video that makes ThriftTheLook's user value and real implementation immediately judgeable.

**Architecture:** The edit follows one complete live run: a clean outfit image, delivery location, budget, streamed processing, eBay results, and an alternative. A short proof section then connects the visible product to the model-provider layer, eBay Browse search, and deterministic delivered-price solver. Offline mode appears only as the zero-key judge fallback.

**Tech Stack:** ThriftTheLook production deployment, a licensed/owned outfit photo, desktop screen capture, Clipchamp free AI voice-over and captions, local MP4 export, unlisted YouTube upload after final approval.

## Global Constraints

- Keep the final public video below 3:00 and include audible narration.
- Use one clean, owned or license-cleared outfit photo; never use a QA screenshot as the hero input.
- Record one fresh successful US live eBay run and retain its own on-screen prices throughout that edit.
- State that displayed totals include shipping to the shown delivery postcode; do not promise tax, duties, availability, or final checkout price.
- The deployed production run currently uses the configured Gemini fallback. Mention GPT-5.6 only after a separately verified GPT-5.6 run or code-level proof; never present Gemini footage as GPT-5.6 footage.
- Do not reveal API keys, account email, Vercel settings, browser history, or private file paths.
- Describe Depop and Vinted only as unavailable/coming-soon integrations; never imply scraping or a live connection.

## Task 1: Create a reliable, truthful hero run

**Files:**
- Create: `temp/video/hero-run-notes.md`
- Create: `temp/video/hero-run.json` (redacted capture metadata only)
- Consume: `https://thriftthelook.vercel.app/`

**Produces:** One live run with a visible completed basket, selected delivery market, total budget, and alternatives. Its prices are the only prices used in the edit.

- [ ] **Step 1: Select the hero image**

Use a vertical, sharp fashion image showing at least two clearly separated visible garments. Confirm it is user-owned, AI-generated by the project, or covered by a source license that permits the demo. Do not use `qa-live-result.png` because it is an application screenshot rather than an outfit photo.

- [ ] **Step 2: Prepare a repeatable search**

Set country to United States, postcode to `10013`, budget to `$120`, and size to `No preference`. This market is supported by eBay image search and shows shipping-aware totals in the product.

- [ ] **Step 3: Record the live run**

Start a fresh browser capture before pressing `Find my look`. Capture the input state, the five progress labels, the result heading, every selected card's delivered total, `Complete look options`, and one expanded `Other options` panel.

- [ ] **Step 4: Validate the capture before editing**

Write the result's marketplace, ZIP, total, selected-item totals, and alternative totals to `temp/video/hero-run.json` without URLs or identifiers. Reject the take if it has a failure state, fewer than two pieces, an obviously irrelevant listing, or an unavailable image.

**Produces:** A 20-28 second technical proof segment whose claims match actual configuration and source.

- [ ] **Step 1: Securely preflight model configuration**

Check only whether the local environment has `OPENAI_API_KEY`, `OPENAI_SOL_MODEL`, and `OPENAI_LUNA_MODEL`; do not print their values. If all are present, run one local controlled outfit request and verify successful model stages. If any are absent or the request fails, do not narrate that the hero run used GPT-5.6.

- [ ] **Step 2: Capture source-level proof**

Record tightly cropped editor shots showing: configured provider selection and structured schemas in `api/llm.py`; eBay Browse/OAuth use in `api/ebay.py`; and deterministic delivered-price assembly in `api/pipeline/assemble.py`. Keep secret-bearing `.env` files and terminal output out of frame.

- [ ] **Step 3: Use the correct narration branch**

If the GPT-5.6 preflight succeeds, say: `GPT-5.6 is the primary structured vision path; the deployed product can fall back to Gemini when that provider is unavailable.` If it does not succeed, say: `The vision layer is provider-configurable, and this deployed live run used its configured Gemini fallback.` Do not merge these statements.

## Task 3: Record narration and edit the 2:20 master

**Files:**
- Create: `temp/video/narration.txt`
- Create: `temp/video/edit-decision-list.md`
- Create: `temp/video/thriftthelook-build-week.mp4`

**Produces:** Captioned 16:9 MP4, target duration 2:20, with spoken audio and no unverifiable claims.

- [ ] **Step 1: Use this base narration**

```text
Most people can save a look. But when they try to thrift it, they are left comparing separate listings, separate shipping, and a budget that stops being real.

ThriftTheLook turns one outfit photo into a complete secondhand basket you can actually buy. I set my delivery location and all-in budget before I search.

This is a live run. The app reads the visible pieces, searches official eBay inventory, and solves for delivered price, not the sticker price. Every result you see includes shipping to the selected ZIP code.

Instead of one random listing, I get a complete look, plus other complete options. I can inspect another shirt choice, and the solver keeps the basket budget-valid.

Under the hood, the vision layer converts the photo into purchasable slots and ranks visual candidates. eBay provides live inventory and shipping context. The final budget decision is deterministic code, so the model never invents the total.

Codex helped us build and test this end-to-end flow. There is also a clearly labelled offline replay, so a judge can run the project with zero keys. ThriftTheLook makes saved fashion actually purchasable.
```

Replace only the model-provider sentence with the approved Task 2 branch. Do not add current prices to narration; the live UI is the source of truth for those.

- [ ] **Step 2: Generate a neutral AI voice and captions**

In Clipchamp, choose a natural English voice at a conversational pace, generate the narration, and create captions. Remove filler pauses but retain enough time to read on-screen UI. Do not use copyrighted music; light royalty-free ambience is optional and must stay beneath the voice.

- [ ] **Step 3: Assemble with intentional pacing**

Use 0.25-0.5 second cuts for setup, real-time playback for at least two streamed progress stages, and a 2-3 second hold on the final basket. Put concise text callouts only where the UI needs help: `Live eBay inventory`, `Shipping included`, `Complete-look alternatives`, and `Deterministic budget solver`.

- [ ] **Step 4: Validate the master**

Confirm duration is 2:20 +/- 10 seconds, captions match narration, audio is intelligible, no key/account data is visible, every current-price claim comes from the same hero run, and the final frame carries the product URL.
